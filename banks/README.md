# Bank logos

Bank logos for financial institutions, organized by country.

## URL convention

```
https://assets.liminal.cash/banks/{country}/{bank_id}.png
```

| URL | Bank |
|-----|------|
| `assets.liminal.cash/banks/gb/nationwide_building_society_gb.png` | Nationwide Building Society (UK) |
| `assets.liminal.cash/banks/gb/barclays_bank_gb.png` | Barclays Bank (UK) |
| `assets.liminal.cash/banks/us/chase_bank_us.png` | Chase Bank (US) |

`country` is lowercase ISO 3166-1 alpha-2. `bank_id` is a stable, lowercase, snake_case identifier that always ends in `_{country_lower}`.

## File requirements

Every bank PNG is paired with a metadata sidecar `{bank_id}.meta.json` next to it.

### `{bank_id}.png`

| Constraint | Value |
|------------|-------|
| Format | PNG |
| Dimensions | 512×512 |
| Background | transparent |
| Aspect | square (letterbox if source is wider/taller, never crop) |
| Max file size | 50 KB |
| Filename | `[a-z0-9_]+\.png`, must equal the metadata `bank_id` |

### `{bank_id}.meta.json`

```json
{
  "bank_id": "nationwide_building_society_gb",
  "name": "Nationwide Building Society",
  "country": "GB",
  "website": "https://www.nationwide.co.uk",
  "swift_codes": ["NWBKGB2L"],
  "source": {
    "url": "https://www.nationwide.co.uk/about-us/media-centre/our-brand",
    "kind": "press_kit",
    "fetched_at": "2026-04-09"
  },
  "verified_by": "jackmarsh",
  "verified_at": "2026-04-09",
  "notes": "Pulled from Nationwide's media centre."
}
```

| Field | Required | Notes |
|-------|----------|-------|
| `bank_id` | yes | Must equal the filename minus extension |
| `name` | yes | Human-readable bank name |
| `country` | yes | Uppercase ISO 3166-1 alpha-2, must equal the directory uppercased |
| `website` | yes | Bank's official website |
| `swift_codes` | optional | Each must be a valid 8 or 11 character SWIFT/BIC; chars 5-6 must equal `country` |
| `source.url` | yes | Where the logo was sourced from. CI fetches it during validation; must return 200. |
| `source.kind` | yes | One of: `press_kit`, `wikimedia`, `bank_website`, `regulatory_filing`, `brandfetch_archived` |
| `source.fetched_at` | yes | ISO date the source was retrieved |
| `verified_by` | yes | One of: `jackmarsh`, `Faizerz` |
| `verified_at` | yes | ISO date verification was performed |
| `notes` | optional | Free-form context for reviewers |

The full schema lives in `scripts/bank.schema.json` and is enforced by CI.

## Sourcing rules (in priority order)

1. **Bank's own press kit / brand asset page** — search `"<bank name> press kit"` or `"<bank name> brand assets"`. Always the first stop.
2. **Wikimedia Commons** — high-quality SVGs with provenance metadata on the file page.
3. **Bank's own website header / favicon** — direct from the bank's homepage.
4. **Regulatory filings** — SEC EDGAR, FCA register, national bank registry.
5. **Third-party logo databases** (last resort) — must use `source.kind: brandfetch_archived` and pass a manual visual confirmation against the bank's actual website.

Never resolve a bank by name lookup against a third-party logo database for the canonical asset. Bank names collide across countries (Nationwide UK Building Society vs Nationwide US Insurance is the canonical example) and the wrong logo is invisible until a human looks.

## Verification

Six layers, each catching a different class of failure:

1. **SWIFT cross-check** — chars 5-6 of every SWIFT code in the metadata must match the directory country. Catches the "wrong country's bank with the same name" failure mode mechanically.
2. **Filename / bank_id structural check** — `bank_id` must end in `_{country_lower}`.
3. **Source URL freshness** — CI fetches `source.url` and fails the PR if it's not 200.
4. **Visual comparison gallery** — `_gallery/index.html` renders every bank logo side by side with the live favicon from the bank's actual website. Reviewers scroll through; mismatches jump out visually.
5. **Hash diff on overwrites** — modifying an existing logo surfaces the pixel-hash diff in the PR comment.
6. **Manual review by code owners** — `verified_by` is restricted to known reviewers; CODEOWNERS gates the merge.

## Gallery

Live at `https://assets.liminal.cash/banks/_gallery/`. Generated by `scripts/build_gallery.py` on every Vercel deploy. The page renders, for every bank in the repo:

- The hosted logo at `https://assets.liminal.cash/banks/{country}/{bank_id}.png`
- The live favicon from `https://{website}/favicon.ico`
- The bank name, country, SWIFT codes, source kind, verified_by

Used during PR review to spot visual mismatches between the hosted logo and the bank's actual brand. Also serves as the public status page — anyone can audit at any time.
